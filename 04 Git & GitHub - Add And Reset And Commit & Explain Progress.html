Git & GitHub [04 Git & GitHub - Add And Reset And Commit & Explain Progress] Concept:

This lesson => Main information about [sending progress] stages via [Git commands] and [cmder]

  1. Main Sending progress via command line and {Cmder} :
    • {Working Directroy} [Still in Local side] =>

      This is the first stage of any project files location , where developer saving his works

      First step ot check the current status of files inside this directory,by using the following command line :
      {git status } => To display the current directory files [before adding files ot hte staging area]
      -> This will display that current directory has files that have been not tracked and need to be added to staging Area

      The command line in this stage is to [add file(s) to the next stage {Staging Area} ] , by using one of following orders =>

      • {git add fileName.Extension } => To add just one file in this directory -> to the Staging Area
      • {git add fileName1.Extension fileName2.Extension fileName3.Extension} => To add Multiply files in this directory -> to the Staging Area
      • {git add * } => To add all of files in this directory -> to the Staging Area

    • {Staging Area} [Still in Local Side] =>

      This is the second stage, where the added files in the ready station {staging Area} and ready to be Commited (all) to the {Local Repo}

      Before Commiting the files to Local Repo , we need to check the current status of files inside this directory, by using the following command line :
      {git status } => To display the current directory files [before adding files ot hte Local Directory] -> This will display that current directory has files that have been not Commited and need to be added to [Local Repo]

      The command line in this step is to {commit} the [ALL file(s) & texutal message] the next step {Local Repo} , by using one of following orders =>

      • {git commit -m "Messag will be sent to the reviewer" } => To send all added Files & Commit message in the staing area (at once) -> to the Local Repo
    • {Sending Back files From Staging step to the Working Directory} -incase of mistake or modification- [Still in Local Side] =>

      This is to send back file(s) that have been added from the [staging Area] -> to the [working Directory] , before being committed to the Local Repo

      The command line in this stage is to {commit} the ALL file(s) the next stage {Local Repo} , by using one of following orders =>

      • {git reset head fileName.extension } => To send back one file back from the [staging area] to the [working Directory]
      • {git reset head fileName1.extension fileName2.extension } => To send back more than one file back from the [staginhg area] to the [working Directory]

      After sending back file(s) to the working directory , we need to check the current status of files inside this directory, by using the following command line :
      {git status } => To display the current directory files [after sending back files to the Working Diretory] -> This will display that current directory has files that have been not Commited and need to be added to [Local Repo] , with out ther file(s) [reset]

    • {Local Repo} => [The Final step in the {local Side}]

      This is the final step of project files in the local side , which will be pushed later to the [Remote local]

      Before pushing commited file(s) form the [local Repo] and push it to the remote repo , we need first to check the current status of files inside this directory after being commmited to local repo , by using the following command line :
      {git status} => To display the current directory files [after Commiting files from {staging Area} to the {Local Repo} ] -> This will display that current directory has files that have been Commited to [Local Repo] , neeeded to be pushed to the [Remote Repo]

git status

// 7- Commiting files form {Staging Area} to the {Local Repo} :
git comnit -m "Message text of will be sent by Dev with files to be read by Reviewer"
